📌 Especificación de los Casos de Uso

Caso de uso Registrarse
ActoresCajero
TipoPrimario
ReferenciasRequisito de seleccionar rol, requisito de validar usuario, CU Iniciar sesión
PrecondicionesEl sistema debe ejecutarse antes de iniciar sesión, debe mostrar los campos de selección de rol, usuario y contraseña, y mostrar el botón Ingresar
PostcondicionesEn caso de datos erróneos, el sistema notifica al usuario advirtiendo con un máximo de 3 intentos
Curso normal (básico) 1. ADMINISTRADOR selecciona su rol e ingresa sus datos
2. CAJERO selecciona su rol e ingresa sus datos
3. SISTEMA valida datos ingresados
AutorDesarrollador
Fecha
Versión1.0
Proposito
Que el usuario ingrese al sistema de acuerdo a su rol
Registrarse
El usuario hace una selección de su rol, luego ingresa sus datos. El sistema los validará de acuerdo a lo ingresado. En caso de datos incorrectos, se notificará al usuario, permitiendo un máximo de 3 intentos.
Caso de uso Validar usuario
ActoresCajero
TipoPrimario
ReferenciasRequisito de validar usuario de acuerdo a su rol, CU Iniciar sesión
PrecondicionesEl sistema deberá tener un botón de iniciar sesión.
PostcondicionesEn caso de datos erróneos, se notificará con un aviso de datos inválidos
Curso normal (básico) 1. USUARIO ingresa datos y da clic en el botón “Iniciar sesión”
2. El sistema verifica los datos
3. El sistema muestra notificación de sesión iniciada
AutorDesarrollador
Fecha
Versión1.0
Caso de uso Selecionar Producto
ActoresCliente
TipoPrimario
ReferenciasRequisito de selección de producto, CU Realizar pedido
PrecondicionesEl sistema deberá mostrar la lista de todos los productos
PostcondicionesEl sistema generará un código de pedido para verificación en caja
Curso normal (básico) 1. CLIENTE ingresa al sistema
2. El sistema muestra la lista de productos
3. El cliente selecciona productos según sus necesidades
AutorDesarrollador
Fecha
Versión1.0
Proposito
Validar a los usuarios de acuerdo a su rol.
Registrarse
El usuario una vez ingresado sus datos al iniciar sesión deberá dar clic en el botón de “Iniciar sesión” y el sistema lo validará de acuerdo a sus datos.
Caso de uso Generar codigo
ActoresCliente
TipoPrimario
ReferenciasValidar datos del cliente, número de cuenta, generar código único. CU Realizar pedido
PrecondicionesEl sistema deberá tener un botón de "Realizar pedido"
PostcondicionesEn caso de datos erróneos, se mostrará aviso de datos inválidos
Curso normal (básico) 1. CLIENTE ingresa datos básicos
2. El sistema verifica datos
3. El sistema muestra notificación de pedido realizado
AutorDesarrollador
Fecha
Versión1.0
Proposito
Selección de un producto y realizar su reserva a través del sistema.
Registrarse
El cliente podrá seleccionar mediante el sistema la lista de productos que ofrece y de acuerdo a sus necesidades.
Caso de uso Verificar codigo
ActoresCajero, Cliente
TipoPrimario
ReferenciasVerificación del código del cliente, generación de factura, CU Realizar pedido
PrecondicionesEl sistema deberá tener un área de texto para el ingreso del código de pedido
PostcondicionesEn caso de datos erróneos, se notificará con un aviso de datos inválidos
Curso normal (básico) 1. CLIENTE presenta el código de pedido
2. CAJERO verifica el código en el sistema
3. El sistema valida el código
4. El sistema actualiza el stock
5. El sistema muestra notificación de “Pedido realizado”
AutorDesarrollador
Fecha
Versión1.0
Proposito
Generar un código de pedido para después hacer un canje en caja.
Registrarse
El cliente deberá facilitar el código de pedido ya generado por el sistema y finalmente el cajero verificará dicho código mediante sistema para su entrega del pedido.
